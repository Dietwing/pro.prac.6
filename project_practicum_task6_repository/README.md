# Базы данных NocoDB для системы НРИ-Помощник

## 1. Назначение работы

В данной работе подготовлена структура базы данных для информационной системы, предназначенной для ведения данных настольно-ролевой игры в локальном формате.

Структура базы данных ориентирована на хранение и обработку следующих видов информации:
- учет пользователей системы;
- учет игровых кампаний;
- учет игровых сессий;
- учет персонажей игроков;
- учет NPC;
- учет боевых столкновений;
- учет участников боя;
- учет бросков кубов;
- учет игровых правил;
- учет заметок и событий;
- учет предметов инвентаря.

## 2. Состав репозитория

В папке проекта находятся:
- CSV-файлы таблиц базы данных;
- файл со схемой сущностей и связей;
- графическое представление модели данных;
- описание логики взаимодействия таблиц.

## 3. Перечень таблиц базы данных

1. Пользователи.csv  
   Содержит учетные записи пользователей приложения, включая роли игрока и гейммастера.

2. Кампании.csv  
   Содержит сведения об игровых кампаниях, игровом мире, системе правил и текущем состоянии кампании.

3. Игровые_сессии.csv  
   Содержит данные о конкретных игровых встречах внутри кампаний.

4. Персонажи.csv  
   Содержит карточки персонажей игроков и основные игровые характеристики.

5. NPC.csv  
   Содержит неигровых персонажей и противников, используемых в кампании.

6. Боевые_столкновения.csv  
   Содержит сведения о боевых сценах, происходящих в рамках игровых сессий.

7. Участники_боя.csv  
   Содержит состав участников каждого боевого столкновения и их текущее состояние.

8. Броски_кубов.csv  
   Содержит журнал бросков кубов, выполняемых в ходе сессий.

9. Правила.csv  
   Содержит справочные правила и игровые описания.

10. Заметки.csv  
    Содержит текстовые заметки, относящиеся к кампании, сессии, персонажу или боевому событию.

11. Предметы_инвентаря.csv  
    Содержит перечень предметов, принадлежащих персонажам.

12. Журнал_событий.csv  
    Содержит последовательность игровых событий, важных изменений и записей по ходу сессии.

## 4. Логика связей между таблицами

Структура базы данных построена по следующей логике.

Пользователи связаны с кампаниями и персонажами.  
Кампании объединяют игровые сессии, персонажей и NPC.  
Игровые сессии включают боевые столкновения, броски кубов, заметки и журнал событий.  
Боевые столкновения содержат участников боя.  
Участники боя могут ссылаться либо на персонажей игроков, либо на NPC.  
Персонажи связаны с предметами инвентаря и журналом игровых действий.  
Заметки и события используются как вспомогательные таблицы для сопровождения хода кампании.

## 5. Практическое применение структуры

Подготовленные таблицы могут использоваться:
- для импорта в NocoDB;
- для разработки локальной базы данных в формате SQLite;
- для визуального проектирования сущностей и их связей;
- для дальнейшей программной реализации системы.

## 6. Использование в NocoDB

Для работы с таблицами в NocoDB необходимо:
1. Создать новый проект.
2. Импортировать CSV-файлы по отдельности.
3. Проверить названия полей и типы данных.
4. Настроить связи между таблицами по логике предметной области.
5. Использовать созданные таблицы как основу базы данных проекта.
